# HackRadar V2 — Vision

## Where V1 landed

V1 works: daily GitHub Actions cron scrapes 25 sources, scores with Gemini 2.5 Flash, emails a top-15 digest. First successful run on 2026-04-08 produced a good-looking email with 15 ranked finds. Proof of concept is solid.

**But V1 has real limits:**

1. **Gemini free tier rate limit hit hard.** 109/452 items scored before 429 RESOURCE_EXHAUSTED kicked in. The daily free quota for gemini-2.5-flash is tiny — ~20 requests per project per day on `generate_content_free_tier_requests`. Every daily run burns through this immediately.
2. **No control surface.** It's a fire-and-forget email. Can't say "show me only stuff from the last 3 days," "rerun with a different focus," "give me more detail on item #4," or "expand search to include X source."
3. **Email is one-shot.** If a find is interesting you have to go dig into it yourself — there's no way to ask follow-up questions, get deeper analysis, or tell the system "more like this."
4. **Scoring is ~good but not great.** Gemini 2.5 Flash is fast/cheap but the hackathon-idea writeups are generic-ish. Better models would produce dramatically better analysis, especially for the high-ranked items where it matters most.
5. **Broken scrapers accumulate silently.** 10/25 sources failed silently in the last run (Stability AI blog 404, OpenAI research 403, HuggingFace models scrape failure, Product Hunt HTML scrape, Chrome Platform, webdev blog, HF datasets, etc). Email doesn't surface this.

## What V2 should be

**A local-first interactive platform, not a cron job.** Runs on Rehan's home PC. The daily email becomes one of many output modes, not the only one.

### Core shifts

- **From daily cron → on-demand + scheduled.** Runs whenever you want, over whatever time window you specify. "Scan the last 3 days." "Scan this specific week." "Re-scan yesterday but focus on audio."
- **From one-way email → interactive UI.** A local web app (Next.js + FastAPI backend? Or just Streamlit/Gradio for speed?) where you can browse results, filter, re-score, expand, ask follow-up questions.
- **From Gemini free tier → local + multi-model.** Use the home PC + free hosted options:
  - Local inference on the home PC (Ollama, llama.cpp, vLLM) for the volume pass (cheap scoring)
  - Hosted premium models for the "deep dive" on top-ranked items (Claude via Claude Max subscription, or free Gemini pro trial, or HuggingFace Inference API)
  - Never hit a paid API for the bulk work
- **From "good-enough" scoring → multi-pass scoring.**
  - Pass 1: cheap local filter (kill the obvious noise, keep maybe 50 candidates)
  - Pass 2: mid-tier scoring on the 50 (local bigger model or free hosted)
  - Pass 3: premium deep-dive on top 10-15 using the best available model (Claude Opus / Gemini 2.5 Pro / GPT-5)
- **Fix and monitor broken scrapers.** Health dashboard in the UI. Sources that fail for >2 runs get flagged visibly.

### Features I want

- **Time window picker.** "Scan last 24h / 48h / 7d / custom range."
- **Source selector.** Turn sources on/off per-run. Group by category (AI research labs, code, dev tools, datasets, etc).
- **Interactive feed.** Browse scored results in a UI with filters (score, category, date, source).
- **Chat-with-item.** Click any item → chat sidebar powered by a premium model. "Tell me more about this." "What's the closest competitor?" "Sketch a hackathon architecture." "What would the demo look like?"
- **"More like this" / "Less like this"** — learn what Rehan actually finds valuable over time.
- **Notes / saved items.** Mark items as interesting, write notes, build a personal library of candidates for the next hackathon.
- **Manual trigger: "Rerun with this focus"** — natural-language override for a single run ("this week I care about anything audio/music related").
- **Deep-dive report on a single item.** Generate a full hackathon blueprint — architecture, tech stack, demo script, risks, setup plan — using a premium model on demand.
- **Source health dashboard.** Which scrapers broke, when, with what error.

### Infrastructure ideas

- **Home PC as the compute host.** Ollama or llama.cpp serving local models. Possibly vLLM if GPU is beefy enough.
- **Models to try locally:**
  - Qwen 2.5/3, Llama 3.3, DeepSeek V3 distills, Gemma 3 — for scoring pass 1
  - Bigger Qwen/Llama variants for pass 2
- **Hosted fallbacks (free tiers):**
  - HuggingFace Inference API
  - Cerebras free tier (insanely fast inference)
  - Groq free tier
  - OpenRouter free tier (has a bunch of free models rotating)
  - Together AI free credits
- **Premium models (sparingly, for top items only):**
  - Claude via Claude Max subscription (Rehan already pays for this)
  - Gemini 2.5 Pro (when free tier allows)
- **Frontend:** Next.js feels right given Rehan's stack. Or Streamlit/Gradio if speed matters more than polish for V2 MVP.
- **Storage:** SQLite for everything (items, scores, seen history, notes, source health). Easy to back up, no server needed.
- **Background scraper:** FastAPI + APScheduler (or just cron on the PC) hits sources in the background, stores to SQLite, UI reads from SQLite.
